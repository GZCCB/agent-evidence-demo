我做了一个真实可运行的本地 Agent 工作流 Demo，用来模拟团队内部对项目笔记、工单摘要和日常进度信息的自动处理。它解决的核心痛点是：人工整理上下文、提炼结论、补行动项和识别风险非常耗时，而且输出格式不稳定，导致协作成本高、响应速度慢。

这个 Demo 的核心逻辑流包括四步：第一步接收原始项目文本；第二步自动提取关键信息并生成摘要；第三步识别待办事项、负责人线索和潜在风险；第四步将结果落成 JSON、运行日志和 Markdown 报告，方便复核、归档和团队共享。虽然当前版本是本地零依赖实现，但整体结构已经按真实 Agent 工作流设计，后续可以很容易接入大模型 API、审批节点和多 Agent 协作链路。

我把它整理成了一个可运行的 GitHub 项目，同时提供浏览器演示页和命令行运行方式。这样既能展示具体成果，也能展示我对 AI Agent 工作流、自动化闭环和可验证交付物的理解。
